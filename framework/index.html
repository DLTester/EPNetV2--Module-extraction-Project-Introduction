IL-Fusion Framework（融合框架）¶

IL-Fusion Framework 是基于 算子（operator）级重构 的跨模态融合执行框架， 用于将 点级融合（Point-level Fusion） 与 网格级融合（Grid-level Fusion） 按照明确的数据流顺序进行组合，形成完整的 LiDAR–Image 融合前向流程。

该框架不关心具体特征提取实现，仅通过接口（interface）调用点云与图像特征， 从而实现 特征提取模块与融合算子的解耦与复用。

框架整体流程¶

IL-Fusion Framework 的前向流程可划分为四个阶段：

  1. 特征提取阶段（Interface）
  2. 调用点云特征提取接口，获得点云特征

  3. 调用图像特征提取接口，获得图像特征图

  4. 点级融合阶段（Point-level Fusion）

  5. 图像特征在点位置进行采样
  6. 点云特征与图像特征分别对齐到融合空间

  7. 基于权重分支进行点级多模态融合

  8. 网格映射阶段（Scatter）

  9. 将点级融合特征按照点到网格索引映射到 BEV 网格

  10. 网格级融合阶段（Grid-level Fusion）

  11. 对 BEV 网格特征进行拼接与卷积融合
  12. 输出最终网格级融合特征

API 文档¶

ILFusionFramework¶

Bases: Module

IL-Fusion Framework（EPNet++ 跨模态融合算子框架）。

该框架严格按照“点级融合 → 网格级融合”的流程组织算子调用， 所有几何 / 逐元素操作通过 torch.ops 调用已注册算子， 所有可学习映射通过 nn.Module 层完成。

特征提取模块通过 Interface 注入，保证 backbone 的可替换性。
